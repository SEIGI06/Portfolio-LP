Structure du dépôt DOC_SISR

Les contenus sont organisés par thématique. Chaque dossier contient un ou plusieurs fichiers Markdown que vous pouvez modifier depuis GitHub ou en local via Git. 

DOC_SISR/
├── linux/                # Commandes, dépannage, scripts
├── proxmox/              # Virtualisation, clustering, sauvegardes
├── active-directory/     # Annuaire, GPO, best practices
├── serveurs-web/         # Apache, HTTPS, reverse-proxy
├── commandes-multi-os/   # Windows, Linux, macOS (gestion, réseau…)
├── pfsense/              # Pare-feu, VPN, politiques de filtrage
└── dns-ha-debian12.md    # Exemple de fiche technique autonome

Workflow de contribution

Travailler en local

1. Cloner le dépôt documentation :

git clone https://github.com/SEIGI06/DOC_SISR.git
cd DOC_SISR
git checkout -b feature/nouvelle-fiche

2. Éditer ou créer des fichiers .md.

3. Commiter et pousser la branche puis créer une Pull Request.

Édition rapide sur GitHub

  • Ouvrir la fiche dans le navigateur.
  • Cliquer sur Edit, apporter les modifications en Markdown.
  • Décrire le changement et valider en créant une Pull Request.
  • Les propositions sont relues avant intégration afin de garder un historique clair.

Aller plus loin : publication automatique

Lorsque la volumétrie sera plus importante, vous pourrez générer une documentation statique (MkDocs, Docusaurus…) depuis ce dépôt et l’héberger via GitHub Pages. Les liens de cette page pointeront alors vers le site généré sans changer la structure actuelle.
